# Vaultalk

> **Real-time, AI-witnessed contract negotiations — built on Stream Chat API**
>
> A chat room where freelancers and clients negotiate freely in natural language. An AI Witness silently reads the conversation, extracts structured terms live, and locks a binding contract + escrow payment the moment both parties type `/agree`.

---

## What to build

Generate the **complete full-stack codebase** for Vaultalk. This is a hackathon project. Prioritize working code over perfect code. Every feature described below must be functional.

## Backend — detailed spec

### `server/index.js`
- Express app on `process.env.PORT || 3001`
- CORS enabled for `http://localhost:5173`
- JSON body parser
- Mount all routes under `/api`
- Serve static client build from `../client/dist` in production

### `server/stream.js`
```js
// Use StreamChat from 'stream-chat' (server-side)
// Export:
//   getClient()         → singleton StreamChat server client
//   createUserToken(userId, userName)  → returns JWT token for that user
//   getOrCreateChannel(roomId, clientId, freelancerId)  → creates/gets a 'messaging' channel
//   sendSystemMessage(channelId, text)  → sends a message with type:'system' and user_id:'vaultalk-bot'
//   getChannelMessages(channelId)       → returns last 100 messages as plain text array
```

### `server/ai.js`
```js
// Uses Anthropic SDK
// Export one function:
//   extractTerms(messages) → calls Claude with the conversation history
//   Returns a JSON object: { price, currency, deliverables[], deadline, revisions, status }
//   where status is: 'negotiating' | 'near-agreement' | 'agreed'
//
// Prompt to use (exact):
// system: "You are a contract term extractor. Read this negotiation chat and extract the current agreed terms.
//          Return ONLY valid JSON with keys: price (number or null), currency (string), deliverables (string[]),
//          deadline (string or null), revisions (number or null), status ('negotiating'|'near-agreement'|'agreed').
//          If a term has not been discussed, set it to null. Never add commentary."
// user: the last 50 messages joined as "Speaker: message\n"
```

### `server/escrow.js`
```js
// In-memory escrow store: Map<roomId, { amount, currency, status, stripePaymentIntentId, clientId, freelancerId }>
// status: 'empty' | 'locked' | 'released' | 'disputed'
// Export:
//   lockFunds(roomId, amount, currency, stripePaymentIntentId, clientId, freelancerId)
//   releaseFunds(roomId)   → marks status 'released', returns stripePaymentIntentId
//   getEscrow(roomId)      → returns escrow object or null
```

### `server/stripe.js`
```js
// Uses 'stripe' npm package in test mode
// Export:
//   createPaymentIntent(amount, currency)  → returns { clientSecret, paymentIntentId }
//   capturePaymentIntent(paymentIntentId)  → captures a previously created intent
//   releasePaymentIntent(paymentIntentId)  → cancels/refunds (for dispute scenario)
// Use capture_method: 'manual' so funds are authorized but not captured until /approve
```

### `server/routes/auth.js`
```
POST /api/auth/join
Body: { name, role, roomId }
- role must be 'client' or 'freelancer'
- userId = slugify(name) + '-' + Date.now()
- Creates Stream user via getClient().upsertUser({ id: userId, name, role })
- Returns: { userId, userName, role, streamToken, roomId }
```

### `server/routes/rooms.js`
```
POST /api/rooms/create
Body: { clientId, freelancerId, roomId }
- Calls getOrCreateChannel(roomId, clientId, freelancerId)
- Sends a system message: "Vaultalk negotiation room opened. The AI Witness is active."
- Returns: { roomId, channelId }

GET /api/rooms/:roomId
- Returns channel info + current escrow status
```

### `server/routes/contract.js`
```
POST /api/contract/generate
Body: { roomId, channelId }
- Fetches last 100 messages from Stream channel
- Calls extractTerms(messages)
- Sends a system message to the channel with the extracted terms as JSON
- Returns: { terms, contractText }
// contractText is a plain English contract generated from the terms

POST /api/contract/export
Body: { roomId, terms }
- Returns a plain text contract document as a downloadable string
- Format: formal contract with parties, terms, date, and "This agreement was witnessed by Vaultalk AI"
```

### `server/routes/payments.js`
```
POST /api/payments/lock
Body: { roomId, amount, currency, clientId, freelancerId }
- Creates Stripe PaymentIntent (manual capture)
- Calls lockFunds(...)
- Sends system message to channel: "Payment of {amount} {currency} locked in escrow."
- Returns: { clientSecret, escrow }

POST /api/payments/release
Body: { roomId }
- Calls capturePaymentIntent with the stored intentId
- Calls releaseFunds(roomId)
- Sends system message: "Payment released to freelancer. Deal complete."
- Returns: { success, escrow }
```

## Slash command handling

In `NegotiationRoom.jsx`, intercept the MessageInput before sending:

```js
const handleSend = async (message) => {
  const text = message.text.trim()

  if (text === '/agree') {
    // 1. Send the message normally so both parties see it
    await channel.sendMessage({ text: '/agree ✅' })
    // 2. Call generateContract
    const { terms, contractText } = await generateContract({ roomId, channelId: channel.id })
    // 3. Open ContractModal with terms + contractText
    setContractData({ terms, contractText })
    setShowContract(true)
    return
  }

  if (text === '/release') {
    if (user.role !== 'freelancer') return // only freelancer triggers
    await channel.sendMessage({ text: '/release ✅ Requesting payment release...' })
    await releasePayment({ roomId })
    return
  }

  if (text === '/dispute') {
    await channel.sendMessage({ text: '/dispute ⚠️ Dispute raised. Evidence preserved.' })
    return
  }

  // Default: send normally
  channel.sendMessage({ text })
}
```

## Important implementation notes

- **Stream channel type**: use `'messaging'` not `'livestream'`
- **Vaultalk Bot user**: create a user with `id: 'vaultalk-bot', name: 'Vaultalk Witness'` on server startup and add it as a member to every channel — this is who sends system messages
- **Terms Panel polling**: poll every 15 seconds BUT also trigger immediately when channel receives a new message of type `'system'`
- **No real money**: Stripe is in test mode, use card `4242 4242 4242 4242` for demo
- **CORS**: server must allow `http://localhost:5173`
- **Stream token**: generate server-side and pass to client — never expose `STREAM_API_SECRET` to browser
- **Channel members**: when creating a channel, add both `clientId`, `freelancerId`, AND `vaultalk-bot` as members
- **Error handling**: every API call must have a try/catch with a visible error state in the UI — judges will break things
